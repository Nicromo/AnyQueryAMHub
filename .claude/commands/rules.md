Напомни мне правила разработки для этого проекта:

1. **Непонятная задача** → остановись и задай вопрос, не угадывай
2. **Короткий код** → никаких абстракций и запасов "на будущее"
3. **Только то, что просили** → не трогай соседний код
4. **Баг-фикс** → сначала тест который падает, потом фикс, потом тест проходит

Эти правила описаны в CLAUDE.md в корне проекта.
