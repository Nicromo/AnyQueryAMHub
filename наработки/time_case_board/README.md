# Case Board (TiMe + Jira)

Локальное приложение: вкладка **TiMe** — Kanban по тредам (`time.tbank.ru`); вкладка **Jira** — список задач по JQL через корпоративный REST API v2 (Bearer), без сохранения тикетов в локальную БД.

**Безопасный режим под VPN:** только `127.0.0.1`, токены в `oauth.env` (не в git). Подробно — [SECURITY.md](SECURITY.md).

## Быстрый старт

1. Скопируйте `oauth.env.example` → `oauth.env`, заполните `OAUTH_CLIENT_ID`, `OAUTH_CLIENT_SECRET`, `OAUTH_REDIRECT_URI` (как в TiMe → Интеграции → OAuth 2.0).  
   Либо задайте **Personal Access Token** в `oauth.env`: `TIME_PERSONAL_ACCESS_TOKEN=...` (тогда OAuth не обязателен).

   Для **Jira** (опционально): `JIRA_BASE_URL`, `JIRA_TOKEN` (тот же Bearer, что в `Invoke-WebRequest`), при необходимости `JIRA_JQL`. Работает только если хост Jira доступен из вашей сети/VPN.

2. Фронт **не обязателен вручную:** при первом запуске `start.sh` / **`start.command`** / **`start.bat`** скрипт сам  
   распакует **`web/dist.zip`** в `web/dist/` (если нет готовой сборки), иначе выполнит `cd web && npm install && npm run build` при наличии **Node/npm**.  
   После правок UI соберите снова: `cd time_case_board/web && npm install && npm run build`.

3. Запуск из каталога `time_case_board` (**без запущенного скрипта сайта не будет**):

   - **macOS / Linux:** в терминале `cd` в папку проекта, затем **`chmod +x start.sh`** (один раз), потом **`./start.sh`**. Либо двойной клик по **`start.command`** (Finder откроет Terminal, поднимется сервер, браузер — когда порт **8790** начнёт отвечать, до ~60 с при первом `pip install`). Скрипты проверяют импорт Python-пакетов после `pip` и при необходимости повторяют установку.
   - **Windows:** двойной клик по **`start.bat`** или из `cmd`: `cd путь\к\time_case_board` → **`start.bat`**. То же: `pip`, проверка импорта, распаковка `dist.zip` или `npm install` / `npm run build`. Окно **не закрывать** — в нём крутится сервер. Перезапуск: **`restart.bat`**.

   **Chrome: «Не удаётся получить доступ», `ERR_CONNECTION_REFUSED`:** на этом компьютере **не запущен** `uvicorn` (закрыли чёрное окно после `start.bat`, не стартовали `./start.sh`, или процесс упал с ошибкой). Запустите скрипт снова и оставьте окно открытым, пока работаете с доской.

   **Окружения Python на разных ОС:** Windows создаёт **`.venv.win`**, macOS/Linux — **`.venv.unix`**. Они лежат рядом в одной папке проекта, но **не перезаписывают друг друга** при синке через iCloud/Dropbox/общий диск. Старую папку **`.venv`** (если осталась от прошлых версий) можно удалить вручную — скрипты её не используют.

   **macOS: нет `bin/python` в venv:** **`rm -rf .venv.unix`** и снова **`./start.sh`** / **`start.command`**. Скрипт сам пересоздаст окружение при необходимости.

   **Windows: «Системе не удается найти указанный путь» / pip install:** удалите **`.venv.win`** и снова запустите **`start.bat`**. Убедитесь, что не подменили `Scripts\python.exe` файлами с Mac.

4. Откройте http://127.0.0.1:8790 (на Windows после `start.bat` вкладка часто откроется сама) — при необходимости «Войти через Time».

**Другой компьютер:** `127.0.0.1` — это всегда «этот» хост. На рабочем ПК сервер поднимется там же (если порт свободен), но **токены и SQLite** лежат локально на той машине: нужен повторный вход в TiMe (или копирование `oauth.env`/БД по правилам ИБ). С другого ПК по сети к вашему `127.0.0.1` подключиться нельзя — только если явно запустить uvicorn на `0.0.0.0` и открыть файрвол (в штатной конфигурации Case Board этого нет).

### Несколько учёток TiMe на одном ПК

- Данные доски хранятся **отдельно на каждую учётку**: `data/board_<id>.sqlite` (файлы не в git).
- Активная учётка: `data/active_time_account.json` (локально, не коммитится).
- **OAuth:** после входа создаётся/открывается БД этого пользователя; в шапке список **«Учётка»** переключает локальную доску. **«Другой логин Time»** — добавить вторую учётку через OAuth.
- **`TIME_PERSONAL_ACCESS_TOKEN` в `oauth.env`:** всегда одна учётка (тот пользователь, чей PAT). Смена учёток через UI отключена — уберите PAT, если нужны несколько OAuth-сессий.
- Старый одиночный файл `data/time_case_board.db` при первом запуске **переименуется** в `board_<ваш id>.sqlite`, если ещё нет `active_time_account.json`.
- Пресеты фильтров и фильтры колонок в браузере хранятся **с привязкой к id учётки** (`localStorage`), чтобы при шаринге репозитория на другой машине не подмешивались чужие сохранённые фильтры.

## Разработка UI

Терминал 1:

```bash
cd time_case_board
source .venv.unix/bin/activate
export PYTHONPATH="$PWD"
python -m uvicorn backend.main:app --reload --host 127.0.0.1 --port 8790
```

Терминал 2:

```bash
cd time_case_board/web && npm run dev
```

Vite проксирует `/api` и `/oauth` на порт 8790.

## API TiMe

Используется REST в стиле Mattermost: `{TIME_BASE_URL}/api/v4/...` и OAuth2 из [документации Time](https://docs.time-messenger.ru/integrations/oauth2_service_provider/).

## Вкладка Jira

- `GET /api/jira/status` — настроена ли интеграция (без утечки токена); при настроенной Jira дополнительно вызывается `GET /rest/api/2/myself` и в ответе приходят `jira_account_id`, `jira_display_name`, `jira_name` — для галочек «последний комментарий мой / не мой» на доске.
- `POST /api/jira/configure` — тело `{"jira_base_url":"https://...","jira_token":"..."}`: записывает `JIRA_BASE_URL` и `JIRA_TOKEN` в **локальный** `.env` в корне приложения (файл в `.gitignore`) и применяет настройки **без перезапуска** сервера. Удобно коллеге после распаковки архива: сначала TiMe OAuth, затем токен Jira через форму во вкладке Jira.
- `GET /api/jira/ping` — проверка `GET /rest/api/2/myself` (кнопка «Проверить Jira»).
- `GET /api/jira/issues` — поиск по JQL; параметры `max_results` (размер страницы API, до 100) и `max_fetch` (сколько задач максимум собрать несколькими запросами, по умолчанию 800, макс. 2000). Без пагинации задачи после первой сотни **не попадут** в UI при `total` больше 100. В ответе есть `loaded_count` и поля **последнего комментария** (`last_comment_author`, `last_comment_author_account_id`, `last_comment_author_name`, если Jira отдаёт `comment` в search).

**Галочки на доске** (сохраняются в `localStorage` вместе с учёткой TiMe, как и колонки Jira): *Скрыть, где последний комментарий мой* — убирает из списка задачи, где последний коммент написан с того же токена (сопоставление по `accountId`, иначе по имени). *Сначала «ждут моего ответа»* — в каждой колонке сверху задачи, где последний комментарий **не** ваш, ниже — где последним писали вы; внутри группы сортировка по времени (комментарий или `updated`).

**Доска по отделам (ключи проектов):** в колонке — один или несколько ключей через запятую: ключ без `*` (например `AAR`) совпадает с проектом `AAR` и с более длинными ключами на тот же префикс (`AARP`, …). Чтобы отделить «узкий» проект от семейства, колонку с **длинным ключом** поставьте **левее** в порядке колонок. Явный **префикс** — буквы + `*` в конце (`AAR*`, `X*`). Колонка с **пустым** списком = «остальное». Если есть `*` в ключе или колонка «остальное», запрос к Jira идёт без `project in`. **Без** «остального» и без `*` в ключах в JQL добавляется `project in (…)` только по перечисленным точным ключам — задач из проектов вроде `AARP` в ответ API тогда не будет, пока не добавите префикс `AAR*` или колонку «остальное». Фильтры по колонке: site ID и текст. Порядок колонок — стрелками / drag. Настройки в `localStorage`.

Если Jira отвечает 400 на JQL, задайте другой запрос в `JIRA_JQL` (например `resolution is EMPTY` вместо `resolution = Unresolved`).

## Колонки и правила

**Один канал в двух колонках:** в SQLite уникальность `(channel_id, root_post_id)` — один тред = одна карточка. При синке карточка привязана к колонке; при повторной загрузке с другой колонки тот же тред может оказаться в ней. Удобнее держать **один канал — одна колонка**. **«Загрузить историю»** у колонки — `POST /api/columns/{id}/history-reset?pages=N`: сброс курсора и подгрузка старых постов; `N` берётся из поля «страниц» в шапке (как у глобальной «История»).

При создании колонки указывается URL канала вида  
`https://time.tbank.ru/tinkoff/channels/channel-name`.

В UI у колонки кнопка **«Фильтр»**:

- **Канал с моими тредами** (`match_self_only`) — в доску попадают только треды, где **корневой пост** написан с учётки, под которой вы вошли в Case Board (TiMe `user_id`). Текст поста **не** проверяется на ваш @логин. После включения сделайте **Синк** и при необходимости **Загрузить историю** по колонке.
- **Только обращения ко мне** (режим саппорта) — по умолчанию включено: в корне треда должен быть `@ваш_логин`, ваш user id в mentions поста, или подстрока из **доп. имён** (`extra_names` в JSON). Иначе тред не попадёт на доску, даже если это форма саппорта.
- **Подстроки автора** — через запятую; среди прошедших фильтр «ко мне» остаются те, где есть хотя бы одна подстрока (часто `@any_1st_line` и т.п.).
- **Только с «Обращение в саппорт»** — для постов в форме саппорта дополнительно требуется эта фраза (DS AMA и др. шаблоны не режутся этим флажком).

**История:** в шапке поле числа + кнопка **«История»** — `POST /api/history/pull` (и дубликат `POST /api/sync/history`) подгружает более старые посты пагинацией `before=` (по ~200 на страницу, до 50 страниц за запрос). После обновления бэкенда перезапусти процесс uvicorn/start.

Новые колонки получают `rules_json` по умолчанию: шаблон `support`, «только ко мне», в `reporter_substrings` подставляются **логин и имя из TiMe `/users/me`** для текущей учётки (не хардкод). В модалке «Фильтр» можно нажать **«Подставить мой логин и имя из TiMe»**.

Фильтр по полю **site ID** ищет и в колонке `site_id`, и **целое число** в тексте корня/треда (чтобы находить `224`, а не подстроку в `1224`).

Под заголовком колонки чекбокс **«я автор треда»** — показывать только карточки, где автор корня совпадает с **текущей** учёткой Case Board (поле `root_author_user_id` в SQLite; подставляется при синке, для старых записей — после очередного синка по каналу).

После смены правил: **«Синк»** подхватывает новые треды; старые «лишние» карточки можно снять кнопкой **«Убрать неподходящие с доски»** в том же модальном окне (или API `POST /api/columns/{id}/prune`).

В `rules_json` также можно править вручную (`PATCH /api/columns/{id}`): `templates`, `require_addressed_to_me`, `match_mentions_me`, `extra_names`, `reporter_substrings`, `support_intro_required`.

## Тесты

```bash
cd time_case_board
source .venv.unix/bin/activate
pytest -q
```

Данные SQLite: каталог `time_case_board/data/` — `board_*.sqlite`, при миграции со старой версии мог остаться `time_case_board.db` (не коммитятся). Jira по-прежнему только через `oauth.env` на этой машине (`JIRA_TOKEN` = ваш пользователь в Jira).
