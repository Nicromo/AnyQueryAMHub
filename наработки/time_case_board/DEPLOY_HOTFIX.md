# Быстрое обновление на машине, где уже крутится Case Board

Сервер отдаёт статику из `web/dist` и код из `backend/`. Чтобы применить правки **без полного клонирования репозитория**, достаточно заменить ограниченный набор файлов и перезапустить процесс (uvicorn / systemd / скрипт запуска).

**Venv по ОС:** Windows — папка **`.venv.win`**, macOS/Linux — **`.venv.unix`** (лежат в корне проекта, при синке одной папки между ПК не затирают друг друга). Обновляя **`start.bat`** / **`start.sh`** / **`start.command`** / **`restart.sh`**, эту схему сохраняйте.

## Вариант A — только фронт (UI: Jira ширина, TiMe превью, подписи)

После `npm run build` в папке `web` на машине с Node **или** скопируй с собранной машины **всю папку** `web/dist/` поверх старой:

| Куда на сервере | Что копировать |
|-----------------|----------------|
| `web/dist/index.html` | из свежей сборки |
| `web/dist/assets/index-*.js` | один файл (имя хэша меняется каждый build) |
| `web/dist/assets/index-*.css` | один файл |

Важно: **`index.html` и пара `assets/*` должны быть из одной и той же сборки**, иначе белый экран или 404 по чанкам.

Перезапуск бэкенда для статики обычно **не обязателен**, если процесс раздаёт папку `dist` с диска при каждом запросе. Если статика вшита или кэшируется агрессивно — перезапусти сервис.

## Вариант B — Jira: подтягивать больше 100 задач (пропавшие тикеты)

Нужны **два** файла бэкенда:

| Файл | Зачем |
|------|--------|
| `backend/jira_client.py` | пагинация `/rest/api/2/search` до `max_fetch` |
| `backend/main.py` | параметр `max_fetch` у `GET /api/jira/issues` |

После копирования — **перезапуск** Python-процесса.

## Вариант C — оба слоя

Скопируй `web/dist/` (как в A) **и** два файла из B, затем перезапуск бэкенда.

## Проверка

- Jira: в шапке доски строка вида «В Jira по JQL: **105**» и совпадение «загружено» с числом карточек (если меньше total — сработал лимит `max_fetch`).
- Слайдер «ширина» в тулбаре на вкладке Jira двигает колонки.
- TiMe: под заголовком треда блок «Последнее в треде» с текстом ответа.

## ERR_CONNECTION_REFUSED / «Сайт не позволяет установить соединение»

На порту **8790 никто не слушает**: не запущен сервер или закрыли окно терминала/`cmd`, где шёл `uvicorn`.

- **Windows:** снова запустите **`start.bat`** из папки проекта и **не закрывайте** чёрное окно, пока пользуетесь доской.
- **macOS:** **`./start.sh`** или **`start.command`** — терминал должен оставаться открытым.

Проверка: в том же окне после старта должны быть строки логов Uvicorn (`Uvicorn running on http://127.0.0.1:8790`).

## Пустой экран на http://127.0.0.1:8790/

1. **Должна быть папка `web/dist/` целиком** — и `index.html`, и подпапка **`assets/`** с `.js` и `.css` (имена с хэшем, например `index-Br1-QkMo.js`). Один только `index.html` без `assets/` даёт белый экран.
2. В **`web/dist/index.html`** в тегах script/link должны быть пути **`/assets/index-….js`** и **`/assets/index-….css`**. Если там **`/src/main.tsx`** — это **исходник** для Vite, не продакшен-сборка: такой файл в `dist` нельзя класть; нужно заново **`cd web && npm run build`**.
3. В браузере **F12 → Network**: обновить страницу. Красные **404** по `/assets/...` = не совпадают файлы или не туда скопировали `dist`.
4. Если видите страницу с текстом *«Фронтенд ещё не собран»* — папки `web/dist` нет или в ней нет `index.html`; выполните сборку из п. 2 README.
