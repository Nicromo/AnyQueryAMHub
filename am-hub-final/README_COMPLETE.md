# AM Hub — Идеальная версия для ежедневной работы аккаунт-менеджеров

## ✅ Что реализовано

### 1. 🧩 Полная интеграция с Merchrules
**Файл:** `merchrules_client.py`

Поддерживаемые эндпоинты:
- `/api/site/all` — список всех сайтов
- `/backend-v2/content` — задачи и контент
- `/api/custom-report` — кастомные отчеты
- `/api/report/agg/{site_id}/global` — агрегированные метрики (выручка, конверсия, RPS, AOV)
- `/api/report/daily/{site_id}/global` — ежедневная динамика
- `/backend-v2/roadmap/comments/latest` — комментарии из роадмапа
- `/backend-v2/meetings/next` — следующая встреча
- `/checkups` — список чекапов
- `/feeds` — список фидов
- `/feed-processing` — статус обработки фида
- `/search-settings` — настройки поиска

**Per-user credentials:** Каждый менеджер может задать свой логин/пароль от Merchrules в настройках.

---

### 2. 🎫 Интеграция с Time (Support)
**Файл:** `time_integration.py`

Возможности:
- Получение открытых тикетов по `site_id` или названию аккаунта
- Фильтрация критических тикетов (priority: high/critical)
- Поиск обращений за период (30/60/90 дней)
- Сводка по поддержке: открытые, критические, последние тикеты

**Требуется:** `TIME_API_TOKEN` в environment variables.

---

### 3. 📹 Интеграция с Ktalk (видеовстречи)
**Файл:** `ktalk_helper.py`

Функции:
- Генерация ссылок на создание комнаты: `/new?topic=Встреча с {client}`
- Поиск записей встреч (артефактов) по названию клиента
- Получение истории встреч за 90/180 дней
- Быстрые кнопки "Начать встречу" и "Найти записи"

---

### 4. 🧠 Pre-Call Brief (Умная подготовка к встречам)
**Файл:** `pre_call_brief.py`

**Что делает:**
1. Собирает данные из всех источников параллельно:
   - Merchrules (задачи, роадмап, фиды, аналитика, чекапы)
   - Time (тикеты поддержки)
   - Ktalk (история встреч)
   - Airtable (QBR события)

2. Выявляет проблемы и алерты:
   - 🔴 Критические тикеты в поддержке
   - ❌ Ошибки обработки фидов
   - 🟡 Просроченные чекапы

3. Генерирует рекомендации через AI (Groq API):
   - 3-5 ключевых рекомендаций для встречи
   - Темы для обсуждения (talking points)
   - Основные риски

4. Форматирует бриф для Telegram (HTML)

**API эндпоинты:**
- `GET /prep/{client_id}/brief` — получить JSON с брифом
- `GET /prep/{client_id}/brief/telegram` — отправить бриф в TG

**UI:** На странице `/prep/{client_id}` добавлены кнопки:
- 🔄 Генерировать бриф
- ✈️ В Telegram

---

### 5. 🚀 Быстрые ссылки на все сервисы

## 🚀 Как использовать

### 1. Запуск приложения
```bash
cd am-hub-final
pip install -r requirements.txt
python main.py
# или
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 2. Подготовка к встрече
1. Откройте `/prep/{client_id}`
2. Нажмите **🔄 Генерировать бриф**
3. Система соберёт данные из всех источников
4. AI сформирует рекомендации
5. Изучите бриф или отправьте в Telegram (**✈️**)

### 3. Начало встречи
1. Нажмите **📹 Новая встреча Ktalk** в блоке быстрых ссылок
2. Откроется комната с предзаполненным названием
3. После встречи запишите ссылку на артефакт в фолоуап

### 4. Проверка проблем клиента
1. Откройте **🎫 Time Support** — проверьте тикеты
2. Откройте **📊 Merchrules Analytics** — изучите метрики
3. Бриф уже выделил критические проблемы 🟡🔴

## 🔧 Расширение функционала

### Добавить новый источник данных
1. Создайте файл `new_source.py` по аналогии с `time_integration.py`
2. Реализуйте класс клиента с методами `get_*`
3. Добавьте вызов в `pre_call_brief.py::generate_pre_call_brief()`
4. Обновите UI в `templates/prep.html`

### Настроить AI-модель
В `pre_call_brief.py::_generate_ai_recommendations()`:
```python
response = await client.chat.completions.create(
    model="llama-3.3-70b-versatile",  # <-- изменить модель
    ...
)
```

## 🐛 Troubleshooting

### "TIME_API_TOKEN not configured"
Добавьте переменную окружения или оставьте пустым — будет работать без тикетов.

### "AI recommendations failed"
Проверьте `GROQ_API_KEY`. Без него бриф будет генерироваться по шаблону.

### Ошибки Merchrules auth
Убедитесь, что менеджер ввёл свои credentials в `/settings` или заданы глобальные `MERCHRULES_LOGIN/PASSWORD`.
