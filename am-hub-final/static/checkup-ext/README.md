# Search Quality Checkup v2 — Chrome Extension

## Быстрый старт

1. `chrome://extensions/` → Режим разработчика → Загрузить распакованное → выбрать папку `ext/`
2. Открыть `lib/config.js` → заполнить API endpoints
3. Готово

## Что заполнить в config.js

```
BACKEND_URL          — URL вашего бэкенда (единая база кабинетов)
BACKEND_TOKEN        — токен авторизации
DASHBOARD_QUERIES_URL — endpoint для запросов чекапа
DASHBOARD_MERCH_RULES_URL — endpoint для мерч-правил
GROQ_API_KEY         — бесплатный ключ Groq (console.groq.com)
CF_ACCOUNT_ID        — Cloudflare account ID (fallback ИИ)
CF_API_TOKEN         — Cloudflare API token
```

## Архитектура

```
ext/
├── manifest.json           — Chrome Extension manifest v3
├── lib/
│   ├── config.js           — ⭐ ВСЕ настройки в одном файле

│   ├── analyzer.js         — Анализ релевантности + эвристики
│   ├── ai.js               — ИИ: Groq (primary) → Cloudflare (fallback)

├── popup/
│   ├── popup.html          — UI расширения
│   └── popup.js            — Логика интерфейса
└── icons/
```

## Логика работы

### 1. Единая база
Менеджер вводит ID кабинета → бэкенд возвращает apiKey, URL, название клиента.
Менеджер НЕ работает с ключами напрямую.

### 2. Калибровка
Сравнивает топ-3 API vs топ-3 на сайте.
- Совпали → API-режим (быстро, автоматически)
- Не совпали → сайт-режим (подсветка расхождений)

Детект товаров на сайте: подход A (автодетект 5 стратегий) → подход B (ручной CSS-селектор).

### 3. Проверка
По каждому запросу:
- Запрос к Diginetica API
- Анализ: stemming + категории + атрибуты
- Автооценка 0-3
- Эвристические рекомендации (бесплатно, мгновенно)
- ИИ-рекомендации для оценок 0-2 (Groq → Cloudflare fallback)

### 4. Рекомендации

#### Эвристики (слой 1):
- Оценка 0: синонимы, редирект, проверка каталога
- Оценка 1: удалить сопутку, пересмотреть мерч-правило, маппинг
- Оценка 2: забустить релевантные, убрать примеси, проверить правило
- Оценка 3: без действий

#### ИИ (слой 2, для оценок 0-2):
Groq (Llama 3.1 70B) → Cloudflare (Llama 3.1 8B) fallback.
Конкретные рекомендации: какой инструмент, что сделать, почему.

### 5. PDF-отчёт
- Шапка: клиент, дата, менеджер, режим
- Сводка: распределение оценок, средний балл
- Проблемные запросы: детали + рекомендации (эвристики + ИИ)
- Таблица всех запросов
- Блок мерч-правил

### 6. Экспорт CSV
Для загрузки в дашборд или Гугл-таблицы.

## Масштаб
- 30 менеджеров × 800 клиентов/мес
- 96 000 запросов/мес → ~35 000 вызовов ИИ
- Groq бесплатно: 1500/день → хватает при распределении
- Cloudflare бесплатно: 10 000/день → с запасом как fallback
